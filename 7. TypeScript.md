# ✅ TypeScript란?

> **TypeScript(타입스크립트)** 는 **Microsoft가 개발한 오픈소스 프로그래밍 언어**로,
> **JavaScript에 정적 타입 시스템(Static Type System)을 추가한 상위 언어(Superset)** 입니다.

즉,

```
TypeScript = JavaScript + 타입(type) + 클래스 기반 OOP 지원 + 컴파일러 검사
```

---

## 🧠 JavaScript vs TypeScript 한눈에 비교

| 구분          | JavaScript        | TypeScript                          |
| ----------- | ----------------- | ----------------------------------- |
| 언어 유형       | 동적 타입             | 정적 타입                               |
| 런타임 오류      | 많음 (컴파일 전 확인 어려움) | 적음 (컴파일 타임에 검출)                     |
| 타입 명시       | X                 | O (`string`, `number`, `boolean` 등) |
| IDE 지원      | 제한적               | 강력한 자동완성, 타입 추론, 리팩토링               |
| 확장성         | 약함                | 대규모 프로젝트에 강함                        |
| 클래스 기반 객체지향 | 지원하지만 미약          | 완전한 클래스 + 인터페이스 지원                  |
| 실행 환경       | 브라우저, Node.js     | 컴파일 후 JavaScript로 변환되어 실행           |

---

## ✅ 예시로 이해하는 TypeScript

### 1. 자바스크립트 코드

```js
function greet(name) {
  return "Hello, " + name.toUpperCase();
}
```

→ 문자열을 기대하지만, 숫자나 undefined가 들어오면 런타임 에러 발생

### 2. 타입스크립트 코드

```ts
function greet(name: string): string {
  return "Hello, " + name.toUpperCase();
}
```

* `name: string`: name은 반드시 문자열
* `: string`: 함수 리턴 타입도 문자열

컴파일 시 타입이 맞지 않으면 즉시 에러 발생합니다.

## ✅ 핵심 기능들 요약

| 기능                           | 설명                             |               |
| ---------------------------- | ------------------------------ | ------------- |
| ✅ 정적 타입                      | 변수, 파라미터, 함수 리턴값에 타입 명시 가능     |               |
| ✅ 타입 추론                      | 타입을 명시하지 않아도 컴파일러가 자동으로 추론     |               |
| ✅ 인터페이스 (`interface`)        | 객체 구조를 명확히 정의                  |               |
| ✅ 제네릭 (`generic`)            | 타입에 유연성과 안정성 동시에 제공            |               |
| ✅ 클래스                        | 객체지향 프로그래밍 문법 정식 지원            |               |
| ✅ 모듈 시스템                     | `import`, `export`를 이용한 모듈화 지원 |               |
| ✅ 유니언 타입 (\`A  B\`)          | 여러 타입 중 하나 허용 |
| ✅ 널 안정성 (`strictNullChecks`) | `null`/`undefined` 오류 사전 방지    |               |

## ✅ 컴파일 과정

TypeScript는 **직접 실행되지 않습니다.**

> `.ts` → **`tsc` 컴파일러로 변환** → `.js` 파일 생성 → 실행

```bash
tsc hello.ts     # → hello.js 생성됨
node hello.js    # 실행
```

## ✅ 설치 및 실행

### 1. 전역 설치

```bash
npm install -g typescript
```

### 2. 버전 확인

```bash
tsc -v
```

### 3. 컴파일

```bash
tsc app.ts
```

---

## ✅ tsconfig.json 설정

`tsc --init` 명령어로 생성.
TypeScript 컴파일러의 설정을 담는 핵심 파일:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
```

## ✅ 실무에서 TypeScript가 왜 중요한가?

| 이유          | 설명                                         |
| ----------- | ------------------------------------------ |
| 🧩 대규모 협업   | 코드의 타입과 구조가 명확해야 다른 개발자와 협업이 가능함           |
| 🐞 오류 사전 방지 | 대부분의 런타임 에러를 **컴파일 타임에 사전에 방지**            |
| 🧠 IDE 자동완성 | IntelliSense, 코드 리팩토링, 점프 기능 강화            |
| 🔍 유지보수 용이  | 타입 기반으로 추적/테스트/확장이 쉬움                      |
| 💡 프레임워크 필수 | Angular, NestJS, Next.js 등 주요 프레임워크가 기본 지원 |
