# 📦 번들이란 무엇인가?

## 1️⃣ 개요: 왜 번들이 필요한가?

웹 초창기에는 HTML, CSS, JavaScript 파일을 별도로 요청해도 문제가 없었습니다. 그러나 **SPA(Single Page Application)**, **모듈 기반 개발**, 그리고 **대형 프론트엔드 프로젝트**가 일반화되면서 수십\~수백 개의 JS/CSS 파일이 생기게 되었고, 이를 그대로 브라우저에서 로딩하면 다음과 같은 문제가 발생합니다.

### ❗ 문제점

* 📡 **HTTP 요청 남발**: 작은 파일 수백 개 요청 → 네트워크 지연
* 🧱 **모듈 의존성 복잡**: 모듈 간 관계를 브라우저가 직접 처리 불가
* 🚨 **브라우저 호환성 문제**: 최신 문법 사용 어려움

이런 문제를 해결하기 위해 등장한 것이 바로 **번들링(Bundling)** 입니다.

---

## 2️⃣ 번들이란 무엇인가?

**번들(Bundle)** 은 여러 개의 자바스크립트(또는 CSS 등) 파일을 하나 또는 몇 개의 **단일 파일로 묶는 과정**을 의미합니다.

> **정의**: “여러 개의 모듈을 병합하여 실행 가능한 단일 파일로 만드는 프로세스 또는 그 결과물”

### 예시

| 번들 전                                | 번들 후        |
| ----------------------------------- | ----------- |
| `app.js`<br>`utils.js`<br>`math.js` | `bundle.js` |
| 30개 모듈                              | 1개 파일로 압축   |

---

## 3️⃣ 번들링이 하는 일

### ✅ 1. 모듈 병합

각 파일에 `import`, `require` 등의 의존성을 분석해 **실행 순서**와 **의존 구조**에 따라 한 파일로 결합합니다.

### ✅ 2. 최신 문법 트랜스파일

ES6+ 문법을 ES5로 변환해 **브라우저 호환성 확보** (Babel 등 사용)

### ✅ 3. 코드 압축 (Minification)

불필요한 공백, 줄바꿈, 주석, 긴 변수명 제거 (Terser 등 사용)

### ✅ 4. 트리 쉐이킹 (Tree Shaking)

사용되지 않는 함수나 코드 제거 → **불필요한 코드 미포함**

---

## 4️⃣ 번들링 전/후 비교

### 📄 예시: 번들 전

```js
// math.js
export function add(a, b) {
  return a + b;
}
```

```js
// app.js
import { add } from './math.js';
console.log(add(2, 3));
```

### 📦 예시: 번들 후 (bundle.js)

```js
function add(a,b){return a+b}console.log(add(2,3));
```

> Terser 등을 통해 난독화, 압축까지 완료된 코드

## 5️⃣ 주요 번들러 도구

| 도구          | 특징                                   |
| ----------- | ------------------------------------ |
| **Webpack** | 강력한 커스터마이징, 플러그인 기반 구조               |
| **Rollup**  | 라이브러리 제작에 최적, 트리 쉐이킹 강력              |
| **Parcel**  | 설정 없이 동작, 빠른 시작 가능                   |
| **Vite**    | 번들보다 모듈 기반 dev-server, 번들은 Rollup 사용 |

---

## 6️⃣ 번들링의 구성 요소

| 용어         | 설명                                 |
| ---------- | ---------------------------------- |
| **Entry**  | 번들링을 시작할 파일 (예: `index.js`)        |
| **Output** | 번들 결과물이 생성되는 파일 (예: `bundle.js`)   |
| **Loader** | JS 외 파일(html, css, 이미지 등)을 처리하는 모듈 |
| **Plugin** | 추가 기능을 수행하는 확장 도구 (압축, 최적화 등)      |

---

## 7️⃣ 번들링이 프론트엔드에 주는 이점

| 항목   | 이점                           |
| ---- | ---------------------------- |
| 성능   | HTTP 요청 수 감소, 초기 로딩 속도 향상    |
| 유지보수 | 모듈화된 구조로 코드 관리 용이            |
| 최적화  | Dead code 제거, 압축으로 파일 사이즈 감소 |
| 생산성  | 최신 문법 사용, 자동 변환/핫 리로딩 지원     |

---

## 8️⃣ 번들링 vs 모듈 시스템

> **모듈 시스템**은 JS 파일 간 **의존성**을 관리하는 방식
> **번들링**은 여러 모듈을 병합하여 **브라우저가 이해할 수 있게** 가공하는 과정

| 항목    | 모듈 시스템               | 번들링             |
| ----- | -------------------- | --------------- |
| 예시    | ES Modules, CommonJS | Webpack, Rollup |
| 기능    | 코드 구조화, 재사용          | 병합, 압축, 최적화     |
| 실행 시점 | 작성 단계                | 배포 전 단계         |

---

## 9️⃣ 코드 스플리팅 (Code Splitting)

**모든 모듈을 한 파일에 다 넣으면 처음 로딩이 무거워질 수 있습니다.**
이 문제를 해결하기 위한 전략이 바로 `코드 스플리팅`입니다.

* 페이지별, 기능별로 JS 파일을 나눠서 **필요할 때만 로드**
* Webpack, Vite 등에서 기본 또는 옵션으로 지원

```js
// 예시: 동적 import
import('./module.js').then(module => {
  module.run();
});
```

---

## 🔚 마무리

**번들링**은 현대 프론트엔드 개발의 필수 과정입니다.
모듈 기반으로 작성된 코드를 실제 브라우저에서 실행 가능하게 만들고,
성능 최적화까지 함께 수행합니다.

### ✅ 핵심 요약

* 번들링은 모듈 → 하나의 실행 파일로 변환하는 과정
* 번들러는 Webpack, Rollup, Parcel, Vite 등이 있음
* 트리 쉐이킹, 압축, 코드 스플리팅 등 최적화 기능 제공
