# ✅ "패키지"란 무엇인가?

## 📦 패키지(package)란?

> 여러 개의 **자바스크립트 코드 파일(JS)**, **설정 파일**, **의존성 정보** 등을 **묶어 배포 가능한 단위**로 만든 것

---

### 📚 비유하자면...

| 용어           | 비유                                  |
| ------------ | ----------------------------------- |
| 패키지          | 완제품처럼 포장된 라이브러리 (ex. axios, lodash) |
| 모듈           | 단일 파일 또는 코드 조각 (보통 `.js`)           |
| 패키지 매니저(npm) | 이 완제품을 전 세계에서 설치하고 관리할 수 있는 도구      |

---

## ✅ 예시로 이해하는 패키지

* 내가 **axios**라는 HTTP 통신 도구를 쓰고 싶다면?

  ```bash
  npm install axios
  ```

  이렇게 설치하면, `node_modules/` 폴더 안에 `axios`라는 **패키지**가 다운로드됨.

* 내부에는 다음과 같은 구조가 있음:

  ```
  node_modules/
    axios/
      index.js
      package.json
      ...
  ```

* 이후 코드에서 바로 사용 가능:

  ```js
  import axios from 'axios';

  axios.get('https://api.example.com/data')
       .then(response => console.log(response.data));
  ```

## ✅ 패키지 개념의 기원은?

### 📦 1. **프로그래밍 세계 전반에서의 '패키지'**

* 자바(Java), 파이썬(Python), 루비(Ruby) 등 **다른 언어들도 '패키지'라는 개념을 오래전부터 사용**
* 의미는 대체로 비슷합니다:

  > **코드를 재사용 가능하도록 묶어, 이름을 붙이고, 배포할 수 있게 만든 단위**

---

## ✅ Node.js와 패키지의 관계

### 📌 2. Node.js 이전의 자바스크립트

* 브라우저 안에서만 동작
* 파일 간 모듈화 어려움 (`<script src="...">` 순서로 의존성 관리)
* **외부 라이브러리 설치나 의존성 관리가 체계적이지 않음**

### 🚀 3. Node.js의 등장 (2009년)

* 자바스크립트를 브라우저 바깥, 서버에서도 실행 가능하게 함
* 서버 사이드 개발에 필요한 도구와 생태계를 정비해야 했음
* 이 과정에서 **npm(Node Package Manager)** 도 함께 등장 (2009년, Isaac Z. Schlueter가 개발)

> 즉, **Node.js는 자바스크립트 생태계에 “패키지”와 “패키지 매니저(npm)” 개념을 정식 도입**한 계기이자 촉진자입니다.

---

## ✅ npm이 자바스크립트 생태계를 어떻게 바꿨나?

| 과거(브라우저만)     | 이후(Node.js + npm)                  |
| ------------- | ---------------------------------- |
| 라이브러리 수동 다운로드 | `npm install`로 자동 설치               |
| 의존성 수동 관리     | `package.json`으로 자동 관리             |
| 코드 재사용 어려움    | 패키지 형태로 누구나 재사용 가능                 |
| 글로벌 생태계 부재    | **npm 레지스트리** 중심의 세계 최대 오픈소스 허브 형성 |

> 현재 npm은 **세계에서 가장 큰 소프트웨어 패키지 저장소**입니다. (2025년 기준 약 250만 개 이상 등록)

---


## ✅ "프로젝트 단위"로의 **Node.js에서 "패키지(package)"**

즉, Node.js에서는 **하나의 프로젝트 폴더 전체**가 **하나의 패키지**이며,
그 패키지의 **중심은 `package.json` 파일**입니다.

---

### ✅ 다시 정의해보자: Node.js에서의 "패키지"

| 구분  | 설명                                  |
| --- | ----------------------------------- |
| 패키지 | **하나의 독립된 Node.js 프로젝트 단위**         |
| 기준  | **`package.json` 파일이 존재하면 패키지로 간주** |
| 형태  | 직접 실행하는 애플리케이션이든, 재사용용 라이브러리든 상관없음  |

> 📌 Node.js에서 **`package.json`이 존재하는 디렉터리는 곧 하나의 패키지이다.**

---

### ✅ 예: 프로젝트 = 패키지

```bash
my-project/
├── package.json  ← 존재 = 이 폴더는 패키지다
├── index.js
├── lib/
│   └── utils.js
├── node_modules/
```

* `my-project/` 전체가 하나의 **Node.js 패키지**
* `npm init`으로 시작한 순간부터 이 디렉토리는 패키지로 간주됨

---

### ✅ 패키지의 두 가지 용도

| 유형            | 설명                                        |
| ------------- | ----------------------------------------- |
| 📦 애플리케이션 패키지 | 직접 실행되는 앱. (예: 웹 서버, CLI 등)               |
| 📦 라이브러리 패키지  | 다른 프로젝트에서 `npm install`로 불러다 쓰는 재사용 라이브러리 |

둘 다 형태는 동일하지만 `package.json` 설정 항목에 따라 목적이 달라짐

---

### ✅ 패키지로 인정받기 위한 최소 조건

```bash
mkdir mylib
cd mylib
npm init -y
```

생성된 구조:

```
mylib/
└── package.json  ✅ 존재
```

이 순간 `mylib/`는 패키지입니다.
npm은 **package.json이 있는 폴더만 "정상적인 패키지"로 인식**합니다.

---

### ✅ 정리 요약

| 질문                    | 대답                                    |
| --------------------- | ------------------------------------- |
| Node.js에서 패키지는 무엇인가요? | **`package.json`이 있는 하나의 프로젝트 폴더 전체** |
| 프로젝트와 패키지는 같나요?       | ✅ Node.js 세계에선 동일한 개념으로 취급            |
| 어떤 조건일 때 패키지가 되나요?    | 최소한 `package.json` 파일이 있어야 함          |

---


## ✅ 결론

> 💡 **패키지라는 개념은 Node.js가 만든 것은 아니지만, 자바스크립트 세계에서는 Node.js가 도입하고 npm이 대중화시켰다**고 보는 게 맞습니다.
